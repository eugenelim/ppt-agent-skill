# Runtime Failure Modes

> 本文件沉淀 Step 4-5 单页生产链路的失败模式。**它定义合同违约和修复顺序，不定义某种审美风格、不压制创新**。
>
> 灵感来源：[sunbigfly/ppt-agent-skills 的 runtime-failure-modes](https://github.com/sunbigfly/ppt-agent-skills/blob/main/references/principles/runtime-failure-modes.md)

---

## 1. 内容未完成类（4 种）

### 1.1 underfill（信息密度过低）

- **触发条件**：页面视觉成立，但 payload 明显不足。
- **失败征兆**：单页信息密度过低，装饰远多于有效信息；卡片留白过多但没有解释性内容。
- **允许创新区**：可以改变构图，但不可以拿留白代替内容合同。
- **修复**：从原始资料里补充数据/案例/解读，填充至策划稿要求的 40-100 字主卡片密度。

### 1.2 support_collapse（支撑卡片塌陷）

- **触发条件**：支撑卡片没有承担解释、比较、证据或上下文。
- **失败征兆**：页面只剩主标题和一两个大数字，缺承托层。
- **允许创新区**：可以改变支撑形式（list / data / process / quote），但必须补足支撑任务。
- **修复**：每个内容页 ≥ 3 张卡片 + ≥ 2 种 card_type + ≥ 1 张 data 卡片。

### 1.3 payload_missing（关键信息缺失）

- **触发条件**：planning 或 HTML 没有完整执行本页应交付的信息。
- **失败征兆**：关键 claim 没有证据、步骤、条件或来源承接。
- **允许创新区**：可以重排内容，但不可以省略必要字段。
- **修复**：对照策划稿 JSON 的 `cards[]` 字段，逐条补齐。

### 1.4 source_overclaim（论断超出资料支持）

- **触发条件**：页面结论强于资料支持力度。
- **失败征兆**：论断看起来完整，但 Step 2 的搜索资料包没有足够证据。
- **允许创新区**：可以强化表达节奏，但不可以夸大事实边界。
- **修复**：弱化结论措辞或补搜索（如"有望成为"改"在 X 行业初见端倪"）。

## 2. 视觉失真类（4 种）

### 2.1 launch_drift（开场风格漂移）

- **触发条件**：封面或章节页把整套 deck 拉向与内容不匹配的发布会语气。
- **失败征兆**：后续信息页难以承接，风格偏置持续放大。
- **允许创新区**：可以做强开场，但必须服从 scene mode（现场/自阅/培训）和受众。
- **修复**：核对 Step 1 的演示场景；如是"自阅文档"，封面应低调而非舞台感。

### 2.2 anchor_overexpansion（锚点过度膨胀）

- **触发条件**：锚点（大数据/大标题）占据全部注意力，支撑内容失去存在空间。
- **失败征兆**：页面单看有冲击，但论证不完整。
- **允许创新区**：可以放大锚点，但不可以吞掉支撑层。
- **修复**：缩小锚点占比至 ≤ 50%；增加支撑卡片或留出阅读路径。

### 2.3 deck_rhythm_clone（节奏克隆）

- **触发条件**：多页使用同构布局、同构装饰、同构锚点关系。
- **失败征兆**：整套 deck 有统一性但没有节奏推进。
- **允许创新区**：可以重复系统，但不可以复制结果。
- **修复**：参考 SKILL.md 的"密度交替"原则——高密度页（混合网格）后跟低密度页（章节封面），形成张弛有度的节奏。

### 2.4 decorative_substitution（装饰替代内容）

- **触发条件**：用材质、光效、分隔线替代真实信息组织。
- **失败征兆**：页面看起来复杂，实际信息承载能力下降。
- **允许创新区**：可以强化材质，但不可以让材质接管结构任务。
- **修复**：删除"为装饰而装饰"的元素；问自己"如果删掉这个装饰，页面信息会损失吗？"，如果不会，删掉。

---

## 3. 修复顺序铁律

任何修复都按以下顺序执行，**不得跳过前两步**：

1. **先补 payload**（内容是地基）
2. **再补 support 与 context**（卡片是承托）
3. **再校正 anchor 的比例与位置**（视觉是叙事）
4. **最后调材质、装饰和局部表现**（装饰是收尾）

不能用装饰调整掩盖内容合同缺失。

---

## 4. 创新自由区

- 创新允许发生在：构图、节奏、装饰组合、锚点表现、材质层次。
- failure mode 只约束违约，不约束风格实验。
- 只要内容合同完整、阅读路径清晰、风格系统稳定，允许页面做高差异化探索。

---

## 5. 自动检测建议

`scripts/smoke_test.py` 通过以下启发式扫描自动检测部分失败模式：

| 失败模式 | 检测启发式 |
|---------|-----------|
| underfill | 单页文字总字数 < 80（封面/章节封面除外）|
| support_collapse | 内容页 cards 数量 < 3 或 card_type 种类 < 2 |
| anchor_overexpansion | 单一卡片占用面积 > 65% |
| decorative_substitution | 装饰类 div 数量 > 文本类 div 数量 × 1.5 |
| deck_rhythm_clone | 连续 3+ 页使用相同的 layout_hint |

其它失败模式（payload_missing, source_overclaim, launch_drift）需要人工或视觉 QA 介入。
