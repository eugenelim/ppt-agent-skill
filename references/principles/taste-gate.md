# 品味闸门与反 AI 感自审（Taste Gate）

> 出片前的最后一道自审：先问"这一页/这一套是不是任何主题都能套出来的默认答案"，再逐条跑删除测试与焦点纪律。灵感源自 Anthropic frontend-design / canvas-design 与"Schematic"线稿图解技能的自审纪律。落点在 Step 5c 每页 HTML 定稿前。

## 为什么需要这道闸门

机械门（lint / 校验脚本）只能证明"没坏"，证明不了"不平庸"。AI 生成的设计会无意识地收敛到几个默认长相 —— 看起来"还行"，但放到任何主题上都一样。这道闸门用来抓住那种"套模板"的收敛。

## 三个"AI 默认长相"—— 自由维度别花在这上面

当 brief 没有钉死某个方向时，避免把创作自由花在这三个已被用滥的默认组合上（除非 brief 明确点名要其中之一）：

1. **暖米底 + 高对比衬线大标题 + 陶土红强调**（"研究手记"味）。
2. **近黑底 + 单一荧光绿/朱红强调**（"科技暗黑"味）。
3. **报纸开版 + 发丝线 + 零圆角 + 密集分栏**（"编辑部"味）。

三者都是合法方向，但都是**默认**而非**选择**。本技能的 `editorial_paper`/`schematic_blueprint` 就分别对应 1 和 3 —— 用它们时要**因为这个主题适合**才用，而不是因为顺手。

## 两遍法：先质疑，再动手

1. **出方案**：为这一套定一个紧凑的 token 系统 —— 配色（4–6 个命名色）、字体（display / body / 必要时 mono 三个角色）、版式概念、**一个招牌动作**（signature move：这套 deck 唯一让人记住的元素）。
2. **质疑方案**：逐条问"如果换一个相似主题，我会不会做出几乎一样的东西？"。凡是读起来像"通用默认"的部分，改掉，并写下改了什么、为什么。确认相对独特后再写 HTML。

## 焦点纪律与减法

- **把大胆用在一个地方**：让招牌动作成为唯一的记忆点，周围一切保持安静克制。
- **强调色是编辑性的，不是信号系统**：accent 落在 1–2 个焦点上；用在 5 个元素上等于没有强调。
- **删除测试**（Schematic 的"最高级动作是删除"）：能删的节点/连线/标签/装饰都删掉 —— 如果删了读者仍然看得懂，就该删。图解节点密度目标约 4/10。
- **香奈儿法则**：出门前照镜子，拿掉一件配饰。

## 出片前品味清单（每页 HTML 定稿前逐条过）

- [ ] **不是默认**：这一页换个主题会不会长一样？若会，说明哪里为这个主题做了专门设计。
- [ ] **一个招牌动作**：这一页（或这一套）的记忆点是什么？只有一个吗？
- [ ] **焦点 ≤ 2**：accent/强调只落在 1–2 个元素上？多了就没重点。
- [ ] **删除测试**：有没有可删的卡片/装饰/连线/标签而不损失信息？
- [ ] **无标题装饰横线**：标题正下方没有那条纯装饰的彩色 rule（eyebrow 标签保留）。见 [design-specs.md](../design-runtime/design-specs.md) 反 AI 感铁律。
- [ ] **正文左对齐**：大段正文不居中（只有标题/金句可居中）。
- [ ] **反差拉开**：最大字号 / 最小字号 ≥ 5 倍；至少 2 种卡片间距。
- [ ] **对比度达标**：文字/图标与底色对比可读（浅底别用浅灰字）。
- [ ] **结构即信息**：编号（01/02/03）、eyebrow、分隔线是在编码真实结构，而非装饰。若内容本不是序列，就别编号。
- [ ] **收尾减一件**：定稿前主动拿掉一个最不必要的元素。
